Computer Science Maths & Tools

On this page I experiment with simple computer science maths using HTML, CSS, and JavaScript. Each tool below takes input, performs a calculation, and explains the idea in clear English.

Binary and decimal converter

Computers work with binary, which uses only 0 and 1. Humans normally use decimal, so this tool converts a decimal number into binary and hexadecimal.

Hint: Try 5, 10, or 42 first and look at how the binary patterns change.

In binary, each position represents a power of 2, such as 1, 2, 4, 8, and 16. For example, the binary number 1010 means 1×8 + 0×4 + 1×2 + 0×1, which equals 10 in decimal. Binary is useful in computers because hardware can represent two clear states, such as on/off or high/low voltage.

2D distance calculator

This tool calculates the distance between two points in 2D space using the formula d = √((x2 - x1)² + (y2 - y1)²).

Hint: Think of the points as coordinates in a game, a map, or a grid.

This formula is important in computer graphics, simulations, robotics, and games. It helps measure how far one object is from another. It can also support pathfinding and movement logic. In simple terms, it tells us the straight-line distance between two coordinates.

Password combinations estimator

This tool estimates how many possible passwords exist based on length and character set size. It uses exponents to show why longer passwords are much harder to guess.

Hint: This is a simplified model. Real password strength also depends on patterns, reuse, and attacker knowledge.
Your result will appear here.

If each position in a password can be filled with N possible symbols and the password length is L, then the total number of combinations is NL. For example, with 26 letters and length 4, there are 26⁴ = 456,976 possible passwords. This shows why increasing password length quickly makes brute-force guessing much harder.
